🔗 How MediatR Discovers Handler & Validator

The magic of automatic wiring through reflection and dependency injection

The Three Files That Work Together

File What It Does Role
LoginCommand.cs Request object (carries data, implements IRequest<AuthResponseDto>) The "message" sent through MediatR
LoginCommandHandler.cs Implements IRequestHandler<LoginCommand, AuthResponseDto> Processes the request, returns response
LoginCommandValidator.cs Implements AbstractValidator<LoginCommand> Validates the request before handler runs

🔌 How They Wire Together (Program.cs)

Step 1: Enable MediatR Auto-Discovery

// In Program.cs (line 24): builder.Services.AddApplication(); // Inside AddApplication() in DependencyInjection.cs (line 12): services.AddMediatR(cfg => cfg.RegisterServicesFromAssembly(typeof(DependencyInjection).Assembly) );

What happens: MediatR scans the HCM.Application assembly and finds ALL classes that implement IRequestHandler<TRequest, TResponse>

Finds: LoginCommandHandler (implements IRequestHandler<LoginCommand, AuthResponseDto>)

Step 2: Enable Validator Auto-Discovery

// In DependencyInjection.cs (line 13): services.AddValidatorsFromAssembly(typeof(DependencyInjection).Assembly);

What happens: FluentValidation scans the assembly and finds ALL classes that inherit from AbstractValidator<T>

Finds: LoginCommandValidator (inherits AbstractValidator<LoginCommand>)

Step 3: Enable Validation Pipeline Behavior

// In DependencyInjection.cs (line 14): services.AddTransient(typeof(IPipelineBehavior<,>), typeof(ValidationBehaviour<,>));

What happens: Registers ValidationBehaviour as middleware that runs BEFORE every handler

🔄 The Request Flow (How They Link Together)

1 Controller sends LoginCommand
await _mediator.Send(new LoginCommand(username, password))
2 MediatR receives the command
MediatR examines the type: LoginCommand
It looks for: IRequestHandler<LoginCommand, AuthResponseDto>
3 MediatR builds the pipeline
Creates: ValidationBehaviour → LoginCommandHandler
(Pipeline behaviors run BEFORE the handler)
4 ValidationBehaviour receives the command
Looks for: IValidator<LoginCommand>
DI container says: "I have LoginCommandValidator!"
Creates instance: new LoginCommandValidator()
Stores in: IEnumerable<IValidator<LoginCommand>>
5 ValidationBehaviour validates the command
Calls: LoginCommandValidator.Validate(loginCommand)
Rules checked:
  • Username.NotEmpty() ✓
  • Username.MaxLength(100) ✓
  • Password.NotEmpty() ✓
  • Password.MaxLength(200) ✓
6 If validation passes...
Calls: next() → passes control to LoginCommandHandler
7 LoginCommandHandler executes
DI injects: IApplicationDbContext, ITokenService, ILogger
Runs: Handle(loginCommand, cancellationToken)
Returns: AuthResponseDto
8 Response returned back through pipeline
AuthResponseDto → ValidationBehaviour → MediatR → Controller → Angular
