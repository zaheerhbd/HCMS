What is HCMS?

HCMS is a Healthcare Case Management System — a web application that helps care teams track patients from their very first visit through discharge. Think of it as a shared digital folder where everyone involved in a patient's care can see what's happening, who's doing what, and what's been done.

The full patient journey — tracked in one place by the whole care team

For Care Coordinators

Register patients, open and manage cases, build care teams, track progress with notes and tasks, upload supporting documents.

For Clinicians

View assigned cases, add clinical notes, review documents, update task status — all without seeing unrelated patient records.

Who Uses It? — Roles

Every person in the system has a role. Your role determines what you can see and do. Think of it like a hospital badge — a student nurse badge opens different doors than a surgeon's badge.

Higher = broader permissions. Each role includes everything below it, plus more.

RoleCan doCannot do
Admin Everything — user management, audit log, all cases, all patients, close/reopen cases —
Supervisor Reassign cases, approve escalations, close cases, supervisor dashboard Cannot manage system users or view the raw audit log
Care Coordinator Register patients, open cases, add team members, create notes, upload documents, schedule appointments Cannot close cases, cannot manage users
Clinician View assigned cases, add clinical notes, view documents, update task status Cannot register patients, cannot change case status
Read Only View patients, cases, and documents Cannot create or modify anything

1 · Logging In

When you open HCMS in your browser you see the login screen. Enter your username and password. The system checks your credentials and issues you a session token — a temporary digital key that identifies you for the duration of your session.

Your session token acts like a hospital visitor badge. It tells every part of the system who you are and what you're allowed to do — without you needing to log in again for each page.

After 5 failed login attempts in a row, the account is temporarily locked for security. An Admin can unlock it.

2 · Patients

A Patient is the person receiving care. All case work happens under a patient record. You must register a patient before you can open a case for them.

Registering a Patient

Only Admin and Care Coordinator can register patients.

  1. Go to Patients in the sidebar and click New Patient.
  2. Fill in demographics: first name, last name, date of birth, gender, contact details, address.
  3. Add insurance information — mark one as Primary.
  4. Click Save. The system assigns a unique Medical Record Number (MRN-2026-00001) automatically.

Finding a Patient

The Patient List page has a search bar at the top. As you type, results update automatically (with a short delay to avoid hammering the server). You can search by name, MRN, or date of birth.

Results are paginated — 20 per page. Use the arrows at the bottom to move between pages.

Click any row to open the Patient Detail page, which shows demographics, insurance records, all cases for that patient, and (in Phase 3) their uploaded documents.

3 · Cases

A Case is an episode of care — a specific reason a patient is being managed by the team. One patient can have multiple cases (e.g., one for their chronic disease management and a separate one for post-surgery follow-up).

Each case has a unique number like CASE-2026-00001.

Opening a Case

  1. Navigate to the Patient Detail page and click New Case, or go to the Cases section and click New Case.
  2. Select the patient (pre-filled if coming from Patient Detail).
  3. Choose a Case Type: Chronic Disease, Post-Surgery, Mental Health, Preventive, or Behavioral.
  4. Write a brief description and set the priority.
  5. Click Create. The case opens with status Open.

Case Status Lifecycle

Every case moves through a defined set of statuses. You cannot skip steps or go backwards arbitrarily — the system enforces valid transitions on both the screen and the server.

Open

Case just created. Awaiting assignment or initial action. Starting point for every case.

In Progress

Active work underway. Team has been assigned. Notes and tasks are being added regularly.

On Hold

Work paused — waiting for an external result (lab test, specialist referral, patient response).

Closed

Care episode complete. Only Supervisor or Admin can close a case. Can be reopened if needed.

Reopened

Case was closed but new information or a complication requires further management.

Closing a case requires Supervisor or Admin role. The system will ask for confirmation before closing.

Care Team

Each case has a Care Team — the specific people responsible for this patient's episode of care. Adding someone to the care team gives them access to the case and alerts them to new notes.

To add a member: enter their User ID and their role on this specific case (e.g. "Lead Coordinator", "Consulting Clinician"). To remove: click the remove button and confirm in the dialog that appears.

Case Notes

Case notes are the running log of everything that happens on a case — clinical observations, phone call summaries, decisions made, follow-up actions.

The 24-hour edit window — You can edit a note for 24 hours after writing it. After that, it becomes permanently read-only. This is a HIPAA compliance rule: clinical records must be tamper-evident. A background job runs every hour to lock expired notes automatically.

4 · User Management

Only Admin can create, edit, and manage user accounts. This section is found in the Admin area of the sidebar.

Creating a User
  1. Go to Admin → Users → New User
  2. Enter username, email, first/last name, and a temporary password
  3. Assign one or more roles
  4. Click Save — user can now log in
Managing a User
  • Change roles — replaces all existing roles atomically
  • Deactivate — blocks login without deleting the account or their history
  • Reactivate — re-enables access instantly
Deactivating a user is always preferred over deleting them. Their audit history, notes, and case records remain fully intact and traceable.

Audit Trail

Every create, update, or delete action on a Patient, Case, Note, or User is automatically recorded in the Audit Log. You don't have to do anything — it happens invisibly in the background.

What's captured
  • Who made the change (User ID)
  • When it happened (UTC timestamp)
  • What changed (old value → new value, stored as JSON)
  • Which record was affected (entity type + ID)
What's protected
  • Audit records can never be deleted or edited
  • No patient names, DOBs, or SSNs in the log — IDs only (HIPAA)
  • Survives even if the user who made the change is later deleted
  • Visible only to Admin role
Analogy: The audit log is like a CCTV system in a hospital medication room. It records every entry automatically — staff don't fill in a paper log, and the footage can't be erased by anyone on site.

Security at a Glance

ProtectionHow it works
PasswordsNever stored in plain text. Hashed with BCrypt (a one-way scramble). Even if someone stole the database, they could not read any password.
Session tokens (JWT)Expire automatically. Must be at least 32 characters long. Stored only in the browser's localStorage.
Role enforcementEvery API endpoint checks your role before doing anything. The UI hides buttons, but the server double-checks every request regardless.
No hard deletesPatients and documents are never truly deleted — they get an "inactive" flag. This preserves history and prevents accidental data loss.
Account lockoutAfter 5 failed logins the account is locked. Prevents brute-force password guessing.
No PHI in logsServer logs contain only entity IDs, never names, dates of birth, insurance numbers, or any other personal health information.

Screen Map — How to Navigate

The app has a persistent sidebar on the left. Every section is one click away.

Breadcrumb Navigation

Every detail page shows a breadcrumb trail at the top so you always know where you are:

  • Patient Detail: Patients › John Smith
  • Case Detail (from patient): Cases › John Smith › CASE-2026-00001
  • Case Detail (from case list): Cases › CASE-2026-00001

Clicking any segment in the breadcrumb navigates back to that level.
