Managed Identity + Key Vault Secrets
~5 min

This lets the API read secrets from Key Vault without any credentials in config files.

8a — Enable Managed Identity on the App Service
  • Go to hcm-prod-api (App Service) → left menu: Identity
  • Under System assigned tab → toggle Status to On → click Save → confirm
  • Copy the Object (principal) ID that appears — you need it next
8b — Grant access to Key Vault
  • Go to hcm-prod-kv (Key Vault) → left menu: Access policies
  • Click + Create
  • Under Permissions: tick Get and List under Secret permissions only → Next
  • Under Principal: search for hcm-prod-api → select it → Next → Create
8c — Add secrets to Key Vault
  • Still in hcm-prod-kv → left menu: Secrets → + Generate/Import
  • Add secret 1:
Secret 1 — Database connection string
NameConnectionStrings--DefaultConnection
ValueServer=hcm-prod-sql.database.windows.net;Database=HcmsDb;User Id=hcmadmin;Password=YOUR_PASSWORD;Encrypt=True;
Replace YOUR_PASSWORD with the SQL admin password you chose in Step 2. The double dash -- in the secret name maps to : in .NET config (e.g. ConnectionStrings:DefaultConnection).
  • Click Create → then + Generate/Import again for secret 2:
Secret 2 — JWT signing secret
NameJwt--Secret
ValueA random string, at least 64 characters long
Generate a strong JWT secret: open browser console on any page and run btoa(crypto.getRandomValues(new Uint8Array(48)).join('')) — copy the result.
8d — Wire Key Vault URI into App Service config
  • Go back to hcm-prod-api → left menu: Environment variables (under Settings)
  • Click + Add for each of these two settings:
App Settings to add
KeyVaultUrihttps://hcm-prod-kv.vault.azure.net/
ASPNETCORE_ENVIRONMENTProduction
ApplicationInsights__ConnectionString(paste the AI connection string from Step 4)
  • Click Apply → Confirm
9
Add GitHub Secrets for CI/CD
~3 min
9a — Get API publish profile
  • Go to hcm-prod-api → Overview page → click Download publish profile in the top toolbar
  • Open the downloaded .PublishSettings file in Notepad → select all → copy all content
9b — Add both secrets to GitHub
  • Go to your GitHub repo → Settings → Secrets and variables → Actions
  • Click New repository secret and add:
GitHub Secrets to add
AZURE_WEBAPP_PUBLISH_PROFILEPaste full XML content from .PublishSettings file
AZURE_STATIC_WEB_APPS_API_TOKENPaste deployment token from Step 7
9c — Trigger CI/CD
  • Push any commit to main branch (or push the existing code if not already pushed)
  • Go to your GitHub repo → Actions tab — watch both workflows run
  • Build & Deploy API deploys the .NET API to hcm-prod-api
  • Build & Deploy Angular deploys the SPA to hcm-prod-swa
