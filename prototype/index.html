FHIR Explorer R4

HL7 FHIR R4 API facade. View patient bundles as they appear from the /fhir/R4/ endpoints.

ℹ️
FHIR R4 Integration: This app exposes a read-only FHIR facade built with the Firely .NET SDK (Hl7.Fhir.R4). Patient data is mapped on-the-fly from SQL Server to FHIR R4 resources — no separate FHIR store needed. Supports Patient, Encounter, Task, Appointment and DocumentReference resources, plus the $everything operation.

Import Patient from FHIR R4

ℹ️
What this does: Paste a FHIR R4 Patient resource from any external system (Epic, Cerner, a referring clinic). The app reads the standard fields, maps them to your internal patient schema, assigns a new MRN, and creates the patient record — no manual data entry needed.

Real app endpoint: POST /fhir/R4/Patient · Validated by Firely .NET SDK · Logged in FhirImportLogs table
Validates the JSON and shows exactly which fields will be mapped before anything is saved.
